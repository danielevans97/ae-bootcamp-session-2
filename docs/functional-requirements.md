# Functional Requirements

## Task Management

1. **Create a task** — The user can add a new task with a title.
2. **Edit a task** — The user can edit the title of an existing task.
3. **Delete a task** — The user can delete a task.
4. **Complete a task** — The user can mark a task as complete or incomplete.

## Task Details

5. **Due date** — The user can add or update a due date on a task.
6. **Priority** — The user can assign a priority level (Low, Medium, High) to a task.
7. **Notes** — The user can add optional notes/description to a task.

## Organization

8. **Sort tasks** — Tasks are sorted by due date (ascending) by default; tasks without a due date appear at the end.
9. **Filter tasks** — The user can filter tasks by status (All, Active, Completed).
10. **Overdue indicator** — Tasks past their due date are visually highlighted as overdue.

## Persistence

11. **Persist tasks** — Tasks are persisted on the backend so they survive page refreshes.
